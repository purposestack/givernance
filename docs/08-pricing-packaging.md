# 08 — Pricing & Packaging

## Packages
1. Starter (small NPO)
   - Core CRM + donations + campaigns
2. Growth
   - Starter + grants + volunteers + impact
3. Pro
   - Growth + SSO/SAML + advanced audit + dedicated support

## Services
- Audit 48h (entry offer)
- Migration package (fixed scope + optional add-ons)
- Training/workshops (admin + operations)

## Pricing logic

### Option A — Per-user subscription (current)
- Platform subscription by active users + transaction bands
- Migration one-off fee
- Premium support SLA optional

### Option B — Commission on processed donations (alternative)
- Low or zero platform fee + commission on donations processed through the platform (e.g., 0.5–1% of donation volume)
- Aligns cost with value created: NPOs pay proportionally to the revenue Givernance helps them collect
- Eliminates the Salesforce pain point of paying for inactive user licenses — field insight from Greg (expert Salesforce + NPO sector)
- Migration one-off fee remains separate
- Premium support SLA optional

### Hybrid approach (recommended to explore)
- Base subscription (reduced) + small commission on processed donations
- Ensures predictable revenue for Givernance while keeping alignment with NPO outcomes
- Commission applies to donations processed via Stripe Connect and postal campaign QR code matching

## Commercial principles
- ROI-visible in first quarter
- No lock-in clauses
- Export-first policy in contract
- Pricing should reflect actual usage and value delivered, not headcount
